Game Development

I am Jayden Kim, and I am a Game Developer

About The Class

This class is about game development through Blender and Unity. Unity is a game engine where you can apply code, physics and other factors to effect the character and its enviroments. At first, I learned about the basic Unity components such as colliders and ridgebody but later moved onto mainly coding in C#.

While using Blender, I learned how to make and texture objects and characters. Because I had prior knowledge on how to use Blender, I started to learn how to make more natural shapes such as people/characters. We did this through mirroring, shrink-wraping and other methods.

My Works

I. Combat System

Programmed algorithms that calculated bullet trajectories, and enemy AI behavior, ensuring a seamless and combat experience. Programming the bullet trajectory in Unity involved implementing accurate physics calculations and incorporating factors such as velocity, gravity, and collision detection to create realistic and dynamic projectile movement. Integrated animations into the programming of bullet trajectory in Unity, synchronizing the firing animations with the projectile's launch and impact, enhancing the visual and immersive experience of the gameplay.

II. World Interaction

Through programming triggers in Unity, we implemented a system where specific events or conditions would activate behaviors in our game. By carefully defining these triggers and their corresponding actions, we were able to create dynamic responses such as enemy attacks, environmental interactions, or character abilities, adding depth and interactivity to the overall gameplay experience.

III. Locomotion

When coding player movement in Unity, we employed input detection to capture user commands, and then used physics-based calculations to translate those inputs into smooth and responsive character motion. By applying forces or adjusting the player's position, we were able to create a dynamic and controllable movement system that allowed players to navigate the game world with precision and fluidity.

IV. Player UI Elements

To enhance the player's shooting experience, we employed UI elements to create a crosshair that accurately represented the aiming position. By tracking mouse or controller input and dynamically updating the crosshair position on the screen, players were able to aim precisely at their targets. Additionally, we incorporated a health UI element that visually displayed the player's remaining health, providing crucial feedback and immersing players in the intensity of combat as they monitored their well-being during gameplay.

V. Enemy AI

To simulate realistic enemy behavior, we programmed the Enemy AI to react to projectile hits by reducing its health based on the damage inflicted. Upon reaching a critical health threshold, a death animation would be triggered, visually indicating the enemy's defeat and adding a satisfying sense of accomplishment for the player.
